# PMD Pro and Finance DPro (Development Sector Project and Financial Management)

## CROSS+WALKRI Compatibility Statement v0.1.0

---

## Summary

PMD Pro (Project DPro) and Finance DPro (FMD Pro) are globally recognized certifications for project and financial management in the development and humanitarian sectors, adopted by USAID, European Union agencies, United Nations agencies, Mercy Corps, Plan International, and many other major donors and implementing organizations. CROSS's three-gate lifecycle structure maps directly to the five phases of the PMD Pro project lifecycle, and WALKRI's measurement instrument requirements satisfy the precision demands of FMD Pro's internal control building block. Grant program operators working in development sector contexts can demonstrate alignment with both certifications through a single CROSS+WALKRI-conformant program design.

---

## The PMD Pro and FMD Pro Frameworks

PMD Pro (pm4ngos.org) is a project management certification developed specifically for NGO and development sector practitioners. Its lifecycle framework covers five phases: project identification and design, planning, implementation, monitoring and evaluation, and closeout. The certification is widely required by major institutional donors of grantee organization staff, making it effectively a prerequisite for managing grants from USAID, EU, and major international NGOs. Unlike generic project management certifications such as PMP, PMD Pro is built around the specific operational context of grant-funded project implementation.

Finance DPro (FMD Pro; fmdpro.org), co-hosted by Humentum and PM4NGOs, is the companion financial management certification. Its four building-block model covers: accounting records, financial planning, financial monitoring, and internal control. FMD Pro is specifically designed for financial managers in NGO and development contexts, where financial management is inseparable from grant reporting obligations and donor compliance. The internal control building block requires that financial monitoring data be traceable to specified sources, a requirement with direct implications for how data collection instruments are designed.

Both certifications are prerequisites or strong preferences for grantee staff at many major institutional donors. Organizations that demonstrate PMD Pro and FMD Pro alignment signal to these donors that their programs are managed according to recognized professional standards. For cohort 3 of the grants ecosystem (grant program operators), alignment with both certifications is often a practical requirement rather than an optional quality marker.

## How CROSS Satisfies the PMD Pro and FMD Pro Frameworks

CROSS's three-gate structure (entry specification gate, progress verification gate, completion gate) maps to the PMD Pro project lifecycle at the phase boundaries where the most consequential decisions are made.

| PMD Pro Phase | CROSS Gate or Mechanism |
|---|---|
| Project identification and design | Entry specification gate: Theory of Change declaration, indicator specification (Fields 1-11), population scope. These are the structured design outputs the PMD Pro framework requires before implementation begins. |
| Planning | CROSS indicator specification Fields 6-11 (baseline, target, timeline, evidence source, collection method, responsible party). Planning outputs are encoded in the entry gate record. |
| Implementation and monitoring and evaluation | Progress verification gate: structured evidence review against specified indicators at defined intervals. This gate produces the monitoring reports PMD Pro requires during the implementation phase. |
| Closeout | Completion gate: final evidence review before the last payment is released. The completion gate record documents that all deliverables have been verified against the original indicator specifications. |

The FMD Pro financial monitoring building block receives structural support from CROSS's concurrent funding disclosure requirement. FMD Pro requires that financial monitoring data accurately reflect the full picture of project funding, including multiple funding sources, restricted versus unrestricted categories, and period-specific allocations. CROSS's concurrent funding disclosure documents all funders, funding amounts, and funding periods for the same program, producing exactly the financial monitoring data that the FMD Pro framework requires to be present in a project's financial records.

FMD Pro's internal control building block deserves separate treatment. Internal control, in the FMD Pro model, requires that financial monitoring data be traceable to specified sources. This means the instruments used to collect financial data during the implementation phase must specify what they are measuring, from whom, and by what method. A financial monitoring report that cannot be traced to a specified data source fails the internal control test regardless of whether the numbers are accurate.

## How WALKRI Complements This Alignment

WALKRI's pre-publication audit requirement directly addresses the traceability demand of FMD Pro's internal control building block. WALKRI requires that every data collection field in a grant program's intake and reporting instruments specify, before any applicant responds: the defined term being measured, the unit of measurement, the baseline condition, the evidence access path, and the population to which the measurement applies. These five requirements ensure that data collected during the PMD Pro implementation phase meets the field-level precision that FMD Pro's internal control building block requires.

The connection is not incidental. A financial monitoring report is only as traceable as the data collection instruments that generated the underlying data. If an intake form asks applicants to describe their outcomes in open-ended prose, the resulting data cannot be traced to a specified source in the FMD Pro sense: there is no defined term, no unit, no baseline against which progress is measured, and no evidence access path that an auditor could follow. WALKRI closes this gap before data collection begins rather than trying to reconstruct precision after the fact.

For development sector program operators managing PMD Pro-aligned projects, WALKRI conformance is the data collection discipline that makes financial monitoring defensible under FMD Pro's internal control standard. The two requirements reinforce each other: PMD Pro specifies that monitoring and evaluation must happen; FMD Pro specifies that the resulting data must be traceable; WALKRI specifies the field-level requirements that make traceability possible.
